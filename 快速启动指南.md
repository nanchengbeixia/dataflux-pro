# DataFlux Pro - 完整项目启动指南

## 🚀 5分钟快速开始

### 步骤1: 环境检查（必须）

```bash
# 检查系统是否满足要求
node environment-check.js
```

**检查内容**:
- ✓ Windows版本 (需要 7 SP1+)
- ✓ Node.js (需要 v16+)
- ✓ npm (需要 7.0+)
- ✓ Microsoft Office 或 WPS (至少一个)
- ✓ .NET Framework 4.6+
- ⚠ Git (可选)
- ⚠ Python (可选)

### 步骤2: 安装依赖

```bash
# 清空npm缓存（遇到安装问题时用）
npm cache clean --force

# 安装所有npm依赖包
npm install

# 如果npm安装很慢，使用淘宝源
npm install -g cnpm
cnpm install
```

**预期输出**:
```
added 500+ packages in 2m
```

### 步骤3: 启动开发服务器

```bash
# 同时启动Vite和Electron
npm run dev
```

**预期行为**:
1. Vite开发服务器启动（http://localhost:5173）
2. Electron窗口打开
3. Vue DevTools工具栏显示
4. 可以修改代码，页面自动刷新

### 步骤4: 开始开发！

现在你可以:
- 📝 修改 `src/renderer/` 中的Vue组件
- 🔧 修改 `src/main/` 中的Electron逻辑
- 🔨 修改 `src/main/ipc-handlers/` 中的IPC处理器
- 💾 修改会自动刷新应用

### 1️⃣ 表格数据分析 (ExcelModule)

**位置**: `src/renderer/components/modules/ExcelModule.vue`

**功能**:
```typescript
// 打开并读取Excel文件
const result = await window.electronAPI.excel.readFile(filePath);

// 分析数据类型和结构
const analysis = await window.electronAPI.excel.analyzeData(
  filePath, 
  'Sheet1', 
  {} // 分析规则
);

// 数据转换（过滤、映射、重命名）
const transformed = await window.electronAPI.excel.transformData(
  data,
  [
    {
      type: 'filter',
      field: 'age',
      condition: 'greaterThan',
      value: 18
    },
    {
      type: 'map',
      sourceField: 'salary',
      targetField: 'annual_salary',
      transform: { type: 'replace', from: '¥', to: '' }
    }
  ]
);

// 保存修改后的数据
await window.electronAPI.excel.saveFile(filePath, transformed);
```

### 3️⃣ 数据关联和填充 (SearchModule) ⭐ 核心功能

**位置**: `src/renderer/components/modules/SearchModule.vue`

**场景**: 将表格2的数据自动填充到表格1

```typescript
// 核心算法：跨表关联查询
const joinResult = await window.electronAPI.search.crossTableJoin({
  sourceTable: table1,        // 源表（要填充的表）
  targetTable: table2,        // 目标表（有数据的表）
  sourceKey: 'employee_id',   // 源表关联字段
  targetKey: 'id',            // 目标表关联字段
  matchFields: [
    'department',
    'position'
  ],                          // 额外匹配字段
  threshold: 0.85             // 匹配度阈值（0.0-1.0）
});

// 查看匹配结果
console.log(joinResult.results);     // 匹配的行
console.log(joinResult.summary);      // 匹配统计

// 数据填充 - 将匹配的数据复制过去
const fillResult = await window.electronAPI.search.fillData({
  sourceData: table2,
  targetData: table1,
  mappings: [
    {
      sourceField: 'salary_account',   // 源表字段
      targetField: 'bank_account'      // 目标表字段
    },
    {
      sourceField: 'phone',
      targetField: 'mobile'
    }
  ],
  joinKey: 'employee_id'  // 关联字段
});

// 结果包含填充日志
console.log(fillResult.data);    // 填充后的数据
console.log(fillResult.fillLog); // 每一行的填充详情
```

**算法说明** (关键！):

1. **Levenshtein距离** - 计算字符串相似度
   ```
   "张三" vs "张 三" = 相似度 0.95 ✓
   "Li Ming" vs "Li M." = 相似度 0.80 ✓
   ```

2. **多字段加权匹配** - 综合多个字段评分
   ```
   员工ID匹配度: 1.0 (完全匹配)
   部门匹配度: 0.9 (模糊匹配)
   位置匹配度: 0.95 (基本匹配)
   综合评分: (1.0 + 0.9 + 0.95) / 3 = 0.95 ✓
   ```

3. **性能优化** - 百万级数据秒速处理
   ```
   创建索引: O(n)
   查询: O(1) 或 O(log n)
   总时间: O(n log n) ≈ 1秒/百万行
   ```

## ❓ 常见问题排查

| 问题 | 症状 | 解决方案 |
|------|------|--------|
| **Electron不启动** | 空白窗口或崩溃 | `npm run dev:vite` + `npm run dev:electron` 分别启动 |
| **Excel无法打开** | "文件不存在"错误 | 检查文件路径，使用绝对路径 |
| **数据乱码** | 显示为 ??? 或乱码 | 自动处理编码，检查文件格式 |
| **关联匹配不准** | 漏匹配或误匹配 | 调整 `threshold` 或增加 `matchFields` |
| **打包文件很大** | exe > 300MB | 使用便携版本，删除不必要依赖 |
| **内存占用过高** | 卡顿 | 分批处理大数据，使用索引 |

## 🎓 进阶话题

### 如何添加新功能？

1. **在后端添加IPC处理器**
   ```typescript
   // src/main/ipc-handlers/custom.ts
   ipcMain.handle('custom:my-feature', async (event, args) => {
     // 实现功能
     return result;
   });
   ```

2. **在预加载脚本暴露API**
   ```typescript
   // src/main/preload.ts
   custom: {
     myFeature: (args) => ipcRenderer.invoke('custom:my-feature', args)
   }
   ```

3. **在前端使用**
   ```typescript
   const result = await window.electronAPI.custom.myFeature(data);
   ```

### 如何优化性能？

1. **使用数据索引**
   ```typescript
   const indexes = await search.createIndex(data, ['id', 'name']);
   ```

2. **限制数据范围**
   ```typescript
   const filtered = data.filter(row => row.year === 2024);
   ```

3. **使用批量查询**
   ```typescript
   const results = await search.batchQuery(indexes, queries);
   ```
